c++八股细节易错点

2025/3/27 学校/考试 c++

为了应付学校c++期末考试而而写，随上课进度更新。

2025.3.24

一、static（复习

1.static 的作用区间是当前文件整个程序的运行期间。static只被创建一次
2.static, const, inline只在一个编译单元起作用（所以inline函数应该定义在头文件中）
3.class中static变量初始化要在class外，且不要再加static；static成员变量不可以在类内初始化，因为static成员变量是属于类的，不是属于对象的，所以不能在类内初始化，否则会报错

class A{
    static int a;
};
int A::a = 0;

//注意，如果是在头文件中定义，会报错，因为头文件会被多次包含，导致重复定义，应当将int A::a = 0;放在.cpp文件中
但是const static变量可以在class中初始化

class A{
    static const int a = 0;
};

总结：static成员变量可以在类外初始化，也可以在类内初始化，但是在类内初始化的static成员变量必须是const类型
4.static成员函数在.h文件中声明，在.cpp文件中定义，定义时不要加static，因为声明时已经加了，否则会报错
5.static成员函数只能访问static成员变量，没有this指针，不用创建具体对象就可以调用（因为static成员函数不属于任何对象，只创建一次，所以不能访问非static成员变量。
6.static成员函数不能被const修饰，因为const成员函数是为了保护对象的成员变量，而static成员函数不属于任何对象，所以没有必要保护

二、namespace

·using namespace 不要写在头文件中，会污染命名空间，不然.cpp一下子全打开了，容易发生冲突
·一个namespace中有多个函数，可以用哪个using哪个，不用全写

namespace A{
    void f1();
    void f2();
}
using A::f1;

三、指针和引用（复习

·指针和引用的区别：指针可以指向空，引用不可以；指针可以改变指向，引用不可以；引用更安全，指针更灵活
·指针要初始化，防止野指针。可以用具体类型的指针初始化void指针，但不能用void指针初始化具体类型的指针，因为void*指针是无类型指针（除非强制类型转换）
·函数返回指针或引用时，不要返回局部变量的指针或引用，因为局部变量在函数结束后会被释放。return static变量或new出来的变量是安全的，因为static变量在程序结束后才会被释放，new出来的变量需要手动释放
·引用一定要初始化为一个对象，不然会报错，但如果是const引用，可以初始化为一个常量（这时编译器会自动创建一个临时变量）

四、拷贝函数

·拷贝函数的作用是在对象被创建时，将一个对象的值赋给另一个对象，本质是一个构造函数，一般格式为X::X(const X& x):a(x.a),b(x.b){}，其中a,b是对象的成员变量
·拷贝构造函数被调用的情况：1.用一个对象初始化另一个对象 2.对象作为函数参数传递 3.对象作为函数返回值(取决于编译器是否优化)
·如果是两个已经创建的对象，直接用=赋值，会调用赋值函数，而不是拷贝函数，赋值函数的格式为X& operator=(const X& x){a=x.a;b=x.b;return *this;}

五、运算符重载

1.+=重载时，返回值要是引用，不然会创建一个临时变量，浪费空间，还可以连续+=，如果不是引用，就会报错，原因是+=返回的是一个临时变量，而临时变量不能作为左值
2.一元运算符重载建议用成员函数，二元运算符重载建议用友元函数

未归类

1.inline函数必须在头文件中定义，否则会报错
