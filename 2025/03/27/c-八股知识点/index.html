10.Name Control

一、static（复习

1.static 的作用区间是当前文件整个程序的运行期间。static只被创建一次

2.static, const, inline只在一个编译单元起作用（所以inline函数应该定义在头文件中）

3.class中static变量初始化要在class外，且不要再加static；static成员变量不可以在类内初始化，因为static成员变量是属于类的，不是属于对象的，所以不能在类内初始化，否则会报错

class A{
    static int a;
};
int A::a = 0;

//注意，如果是在头文件中定义，会报错，因为头文件会被多次包含，导致重复定义，应当将int A::a = 0;放在.cpp文件中
但是const static变量可以在class中初始化

class A{
    static const int a = 0;
};

总结：static成员变量可以在类外初始化，也可以在类内初始化，但是在类内初始化的static成员变量必须是const类型

4.static成员函数在.h文件中声明，在.cpp文件中定义，定义时不要加static，因为声明时已经加了，否则会报错

5.static成员函数只能访问static成员变量，没有this指针，不用创建具体对象就可以调用（因为static成员函数不属于任何对象，只创建一次，所以不能访问非static成员变量。

6.static成员函数不能被const修饰，因为const成员函数是为了保护对象的成员变量，而static成员函数不属于任何对象，所以没有必要保护

二、namespace

1.using namespace 不要写在头文件中，会污染命名空间，不然.cpp一下子全打开了，容易发生冲突

2.一个namespace中有多个函数，可以用哪个using哪个，不用全写

namespace A{
    void f1();
    void f2();
}
using A::f1;

3.scope resolution：作用域解析符”::”

4.命名空间可以嵌套

namespace A{
    namespace B{
        void f1();
    }
}

using namespace A::B;

5.命名空间可以重命名

namespace A{
    namespace B{
        void f1();
    }
}
namespace C = A::B;

6.命名空间可以分开定义

namespace A{
    void f1();
}
namespace A{
    void f2();
}

11.References & the Copy-Constructor

一、指针和引用（复习

1.指针和引用的区别：指针可以指向空，引用不可以；指针可以改变指向，引用不可以；引用更安全，指针更灵活

2.指针要初始化，防止野指针。可以用具体类型的指针初始化void指针，但不能用void指针初始化具体类型的指针，因为void*指针是无类型指针（除非强制类型转换）

3.函数返回指针或引用时，不要返回局部变量的指针或引用，因为局部变量在函数结束后会被释放。return static变量或new出来的变量是安全的，因为static变量在程序结束后才会被释放，new出来的变量需要手动释放

4.引用一定要初始化为一个对象，不然会报错，但如果是const引用，可以初始化为一个常量（这时编译器会自动创建一个临时变量）

二、拷贝函数

1.拷贝函数的作用是在对象被创建时，将一个对象的值赋给另一个对象，本质是一个构造函数，一般格式为X::X(const X& x):a(x.a),b(x.b){}，其中a,b是对象的成员变量，a和b要写在初始化列表中

2.拷贝构造函数被调用的情况：1.用一个对象初始化另一个对象（一定注意是初始化！这是一个正在创建中的对象而不是已经有的对象！像b=a两个已有对象就会调用赋值函数而不是拷贝函数） 2.对象作为函数参数传递 3.对象作为函数返回值(取决于编译器是否优化)

3.如果是两个已经创建的对象，直接用=赋值，会调用赋值函数，而不是拷贝函数，赋值函数的格式为X& operator=(const X& x){a=x.a;b=x.b;return *this;}

4.如果类中有指针成员变量，拷贝构造函数和赋值函数要深拷贝，否则会出现浅拷贝的问题，导致内存泄漏或重复释放内存。深拷贝的格式为X::X(const X& x):a(new int(*x.a)),b(x.b){}，其中a是指针成员变量，b是普通成员变量
（浅拷贝导致重复释放例子可见3.26作业：英文教材ex11_17）

class A{
    int* a;
    int b;
    A(int x, int y):a(new int(x)),b(y){} // 构造函数
    A(const A& x):a(new int(*x.a)),b(x.b){} // 拷贝构造函数
    A& operator=(const A& x){ // 赋值函数
        if(this != &x){ // 防止自赋值
            delete a; // 释放原来的内存
            a = new int(*x.a); // 深拷贝
            b = x.b;
        }
        return *this;
    }
    ~A(){delete a;} // 析构函数
};

5.编译器自动生成的拷贝构造是“逐个成员变量”调用拷贝构造的。如果对象中存在另一个类对象的成员变量，例如Class B中有Class A的对象，那么编译器会自动调用Class A的拷贝构造函数来拷贝Class B中的Class A对象。

12.Operator Overloading

一、运算符重载

1.各类型运算符重载函数实例

1）四则运算符
class A{
    int a;
public:
    A(int x):a(x){}
    A operator+(const A& x){ // 二元运算符重载
        return A(a + x.a);
    }
};

或

class A{
    int a;
public:
    A(int x):a(x){}
    friend A operator+(const A& x, const A& y){ // 友元函数重载
        return A(x.a + y.a);
    }
};
2）赋值运算符=
class A{
    int a;
public:
    A(int x):a(x){}
    A& operator=(const A& x){ // 赋值运算符重载
        a = x.a;
        return *this; // 返回引用
    }
};
3）与等号结合的运算符
class A{
    int a;
public:
    A(int x):a(x){}
    A& operator+=(const A& x){ // +=重载
        a += x.a;
        return *this; // 返回引用，可以连续+=（可以做左值）
    }
};
4）自增自减运算符
class A{
    int a;
public:
    A(int x):a(x){}
    A& operator++(){ // 前置运算符重载
        ++a;
        return *this;
    }
    A operator++(int){ // 后置运算符重载
        A temp = *this; // 先保存当前对象的副本
        ++a;
        return temp; // 返回副本
    }
};
5）<<和>>
class A{
    int a;
public:
    A(int x):a(x){}
    friend ostream& operator<<(ostream& os, const A& x){ // 重载<<运算符
        os << x.a;
        return os; // 返回ostream对象的引用，可以连续<<
    }
    friend istream& operator>>(istream& is, A& x){ // 重载>>运算符
        is >> x.a;
        return is; // 返回istream对象的引用
    }
};

注意const什么时候该用什么时候不用

6）[]
class A{
    char* a;
public:
    A(const char* x){
        a = new char[strlen(x) + 1];//+1是为了存放'\0'
        memset(a, 0, strlen(x) + 1);
    }
    ~A(){
        delete[] a;
    }
    char& operator[](int i){ // 重载[]运算符
        return a[i];
    }
};
7）->和()在数据类型转换中的重载应用
class A{
    int a;
    public:
    A(int x):a(x){}
    operator int*(){ // 重载->运算符
        return &a;
    }
    operator int() const{ // 重载()运算符
        return a;
    }
};

2.所有一元运算符重载为member;
=,(),[],->,->* 必须重载为member;
跟等号结合的二元运算符重载为member;
其他二元运算符重载为friend;

3.不能重载的运算符：., ::, sizeof, ?:

4.重载不改变运算符的优先级，语法，结合性。例如，++比.运算优先级低，比如你还有一个print()用来输出a的值，那么可以直接写a++.prin()，但不能直接写++a.print()！

5..返回引用还是副本取决于你重载的运算符的性质，比如+=，++，–等运算符返回引用，因为是对对象本身的操作，而+，-等运算符返回副本，因为是对对象的值的操作

13 Dynamic Object Creation

一、new和delete

1.detete释放的是指针指向的内存，而不是指针本身，所以delete ptr; 之后ptr还是存在的，只是指向的内存被释放了，ptr仍然可以使用，但不建议使用，因为ptr指向的内存已经被释放了，可能会导致未定义行为。所以，在 delete ptr; 之后写 ptr = nullptr; 是一个良好的编程习惯。

int* ptr = new int(10);
delete ptr;
ptr = nullptr; // good practice

2.数组情况用delete[]，单个对象用delete

int* ptr = new int[10]; // 动态分配数组
delete[] ptr; // 释放数组

3.对于有指针成员变量的类，一定要自己定义拷贝构造函数和赋值函数，否则会出现浅拷贝的问题，导致内存泄漏或重复释放内存。深拷贝的格式为X::X(const X& x):a(new int(*x.a)),b(x.b){}，其中a是指针成员变量，b是普通成员变量

Inheritance & Composition

1.从基类继承来的函数，如果子类没有重写，那么调用的是基类的函数，里面的变量是基类的变量，而不是子类的变量。

2.基类必须在派生类之前定义，否则编译器会报错，不能只声明不定义。

3.继承与组合的本质区别：继承是is-a关系，组合是has-a关系。

4.派生类对象构造顺序：先构造基类，再构造派生类。注意不是按初始化列表顺序，而是派生类中的基类继承顺序和成员变量声明顺序。

6.不能继承到派生类的函数：Constructor, Destructor, Operator Overloading

7.继承的三个关键词：public, private, protected

8.通过private继承后，不能直接使用基类成员，可以间接访问。

9.如果有两个基类有名字一样的成员函数，解决办法有：

  • 作用域限定符
  • 重新定义基类成员函数
  • 虚基类（当派生类从多个基类派生，而这些基类又有共同基类，则一开始就将共同基类设为虚基类，这样派生类就可以继承该基类的成员函数）

10.派生类定义了名字相同的数据成员，则会隐藏基类的同名数据成员，如果需要访问基类的同名数据成员可以用作用域限定符。

Polymorphism & Virtual Functions

1.polymorphism与overloading的区别：overloading是编译时重载，polymorphism是运行时多态。
- overriding：子类重新定义基类中的虚函数，使得派生类对象调用该函数时，调用的是子类的函数。如果派生类没有重写该函数，那么调用的是基类的函数。
- overloading：同一个作用域中，可以有多个同名函数，只要参数列表不同即可。

2.虚函数的核心目的是通过基类访问派生类定义的函数，实现运行时多态。

3.虚函数的声明：virtual 返回值类型 函数名(参数表) const;
虚函数不能是static的，因为static函数属于类本身，与对象无关，不能被多态调用。

4.虚函数只需要在声明时加上关键字virtual，在类外定义时不加。

5.polymorphic条件：
- 基类中有virtual函数
- 派生类继承为public
- 派生类中函数参数列表与基类中函数参数列表一致
- 基类通过指针或引用指向派生类对象，调用虚函数
cpp //A是基类，B是派生类 B b; A* pa = &b; //或A &ra = b;但不能是A a = b,因为这样在运行之前就已经确定了对象的类型 pa->func(); // 调用的是B的func函数 //如果B中有A中没有的函数，那么pa无法调用，因为此时pa是A类对象

6.abstract class：抽象类，不能实例化，只能作为基类被继承，不能创建对象，只能作为接口使用。包含纯虚函数的类就是抽象类。但可以声明抽象类引用或指向抽象类的指针。

class Shape{
public:
    virtual double area() = 0; //纯虚函数
    //如果要带参数
    virtual double perimeter()(double) = 0; //纯虚函数
};

7.派生类会隐藏基类的同名成员函数，如果基类中Overloading了两个virtual f()，在派生类中Overriding其中一个，那么基类中的另一个就不能再被调用；或者Overloading了f()，那么基类中的f()就不能再被调用。除非用作用域限定符。

8.构造函数不能是虚函数，因为构造函数不能被多态调用。

9.析构函数可以是虚函数，用虚函数更安全。在析构时，会先调用派生类的析构函数，再调用基类的析构函数。

10.向上类型转换是安全的，但向下类型转换是不安全的。

11.虚函数表的大小是4个字节，存放的是虚函数的地址。虚函数表是一个数组，存放的是虚函数的地址。每个类都有一个虚函数表，存放该类的虚函数的地址。每个对象都有一个指针，指向该对象所属类的虚函数表。

12.如果在基类中声明了一个Base* clone()，返回指向当前对象副本的指针return new Base(this)，然后在两个派生类A, B中分别重写Base clone()（注意要写Base* clone而不是A* clone或B* clone，不然就只能return new A或B而不能return new Base了），此时如果返回值写return new A(*this)或B(*this)，那么调用clone()时返回的是派生类的对象；如果写return new Base(*this)，那么调用clone()时返回的是基类的对象。

Templates 模板

1.当几个函数或者类中只有部分代码不同时（如参数类型不同），可以用模板来实现代码的重用。

2.函数模板基本语法：

template<typename T1, typename T2>//类型参数列表
T1 Max(T1 a, T2 b){
    return (a > T1(b)) ? a : T1(b); //类型转换
}

int main(){
    cout << Max('a', 3) << endl; 
    return 0;
}

3.函数模板也可以重载

4.类模板基本语法：

template<typename T1, typename T2>
class Test{
    public:
         Test(T1 x, T2 y):a(x),b(y){};
         void print();
    private:
        T1 a;
        T2 b;
};

void Test<T1, T2>::print(){ //成员函数在类外定义必须要用函数模板
    cout << "a = " << a << " b = " << b << endl;
}

int main(){
    Test<int, char> t(10, 'a');
    t.print();
    return 0;
}

5.从模板类派生出非模板类：

template<typename T>
class Base{
    //
};
class Derived : public Base<int>{ //类型参数应实例化
    //
};

6.从类模板派生出类模板：

template<typename T>
class Base{
    //
};

template<typename T>
class Derived : public Base<T>{ //类型参数应实例化
    //
};

7.例：冒牌排序的函数模板实现

template<typename T>
void BubbleSort(T arr[], int n){
    for(int i = 0; i < n-1; i++){
        for(int j = 0; j < n-i-1; j++){
            if(arr[j] > arr[j+1]){
                T temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
        }
    }
}

int main(){
    int arr[] = {5, 2, 8, 3, 1, 9, 6, 7, 4};
    int n = sizeof(arr)/sizeof(arr[0]);
    BubbleSort(arr, n);
    for(int i = 0; i < n; i++){
        cout << arr[i] << " ";
    }
    return 0;
}

2.不能以ostream对象作为参数传递给函数，因为ostream对象是一个流对象，不能被复制，只能被引用或指针传递
```cpp
void print(ostream& os){ // 正确

3.存储字符串的动态数组：

class MyString {
    char* data;  // 存储字符串的动态数组
    int length;  // 字符串长度

public:
    // 构造函数
    MyString(int len) : length(len) {
        data = new char[length + 1]; // 多分配1个字节存放 '\0'
        memset(data, 0, length + 1); // 初始化为空字符
    }

    // 析构函数
    ~MyString() {
        delete[] data;
    }
//str.data 是一个 char* 指针，指向动态分配的字符数组。在 C++ 中，std::ostream（比如 cout）的 operator<< 具有针对 char* 的特化重载，它会 将 char* 解释为 C 风格字符串（以 \0 结尾的字符数组），并从指针开始，逐个输出字符，直到遇到 \0 结束。所以直接cout <<str.data; 就可以输出字符串了。
    friend std::ostream& operator<<(std::ostream& os, const MyString& str) {
        os << str.data; // 输出字符串
        return os;
    }
};

void f()const{};//这种写法只能修饰成员函数，表示该函数不能修改对象的成员变量
//不能修饰非成员函数，因为非成员函数没有this指针，没有意义
